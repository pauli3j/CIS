01

Inventory and Control of Enterprise Assets

IG1 IG2 IG3
Total Safeguards 5
Asset Type Devices
Security Function Identify, Protect, Detect

Overview 📋

Actively manage (inventory, track, and correct) all enterprise assets (end-user devices, including portable and mobile; network devices; non-computing/Internet of Things (IoT) devices; and servers) connected to the infrastructure physically, virtually, remotely, and those within cloud environments, to accurately know the totality of assets that need to be monitored and protected within the enterprise.

Why is this Control critical? 🤔

  • Enterprises cannot defend what they do not know they have
  • Proper asset management helps identify critical data and systems
  • External attackers continuously scan for unprotected assets
  • New assets without secure configuration can be exploited
  • Unidentified assets may have weak security configurations
  • Additional connecting assets need identification
  • Dynamic nature of portable devices makes inventory challenging
  • Cloud environments and virtual machines add complexity to tracking
  • Complete inventory supports incident response investigations

Procedures and Tools 🔧

  • Requires both technical and procedural actions
  • Links to business governance through data/asset owners
  • Can use enterprise products or security tools for inventory
  • Multiple data sources needed for accurate inventory:
    • Regular network scanning
    • Cloud portal data
    • Active Directory logs
    • SSO and MFA systems
    • VPN logs
    • IDS/DPI systems
    • MDM tools
    • Vulnerability scanning results

Safeguards 🔒

1.1 Establish and Maintain Detailed Enterprise Asset Inventory

  • Include network address, hardware name, owner, department
  • Cover physical, virtual, remote, and cloud assets
  • Update bi-annually or more frequently

1.2 Address Unauthorized Assets

  • Process to address unauthorized assets weekly
  • Options: remove, deny remote access, or quarantine

1.3 Utilize an Active Discovery Tool

  • Daily or more frequent execution
  • Identify connected assets

1.4 Use DHCP Logging to Update Asset Inventory

  • Weekly or more frequent updates
  • Use logs from all DHCP servers

1.5 Use a Passive Asset Discovery Tool

  • Weekly or more frequent scans
  • Update inventory based on findings
02

Inventory and Control of Software Assets

IG1 IG2 IG3
Total Safeguards 7
Asset Type Applications
Security Function Identify, Protect, Detect

Overview 📋

Actively manage (inventory, track, and correct) all software (operating systems and applications) on the network so that only authorized software is installed and can execute, and that unauthorized and unmanaged software is found and prevented from installation or execution.

Why is this Control critical? 🤔

  • Complete software inventory is critical for preventing attacks
  • Attackers scan for vulnerable versions of software to exploit
  • Without inventory, enterprises cannot determine vulnerable software or licensing violations
  • Software inventory helps guard against known attacks until patches are released
  • Can help mitigate zero-day exploits through temporary measures
  • Helps identify unnecessary security risks from unused software
  • Critical to inventory, understand, assess, and manage all software

Procedures and Tools 🔧

  • Implement allowlisting using:
    • Commercial allowlisting tools
    • Anti-malware suite features
    • Operating system built-in tools
  • Use standardized application names (Common Platform Enumeration)
  • Leverage Security Content Automation Protocol (SCAP)
  • Modern endpoint security suites often include:
    • Anti-malware
    • Anti-spyware
    • Personal firewall
    • Host-based IDS/IPS
    • Application allowlisting

Safeguards 🔒

2.1 Establish and Maintain a Software Inventory

  • Document title, publisher, install date, business purpose
  • Include URL, app store, versions, deployment mechanism
  • Update bi-annually or more frequently

2.2 Ensure Authorized Software is Currently Supported

  • Monthly review of software support status
  • Document exceptions for necessary unsupported software

2.3 Address Unauthorized Software

  • Remove or document exceptions
  • Monthly review cycle

2.4 Utilize Automated Software Inventory Tools

  • Automate discovery and documentation
  • Enterprise-wide implementation

2.5 Allowlist Authorized Software

  • Use technical controls to restrict execution
  • Bi-annual reassessment

2.6 Allowlist Authorized Libraries

  • Control loading of software libraries
  • Block unauthorized libraries

2.7 Allowlist Authorized Scripts

  • Use digital signatures and version control
  • Block unauthorized scripts
  • Bi-annual reassessment
03

Data Protection

IG1 IG2 IG3
Total Safeguards 14
Asset Type Data
Security Function Identify, Protect, Detect

Overview 📋

Develop processes and technical controls to identify, classify, securely handle, retain, and dispose of data.

Why is this Control critical? 🤔

  • Data is no longer contained within enterprise borders:
    • Cloud storage
    • Remote work devices
    • Partner sharing
    • Online services
  • Privacy rules are complex for multi-national enterprises
  • Data exfiltration may go unnoticed without monitoring
  • Threats include:
    • Physical theft of devices
    • Network attacks
    • Service provider breaches
    • SCADA system compromises
  • Most data loss occurs from poor management and user error
  • Encryption is crucial for data protection and compliance

Procedures and Tools 🔧

  • Develop comprehensive data management process including:
    • Data management framework
    • Data classification guidelines
    • Protection requirements
    • Handling procedures
    • Retention policies
    • Disposal procedures
  • Establish data breach process integrated with:
    • Incident response plan
    • Compliance requirements
    • Communication plans
  • Create data classification scheme (e.g., "Sensitive," "Confidential," "Public")
  • Develop data inventory mapping software access levels
  • Implement network segmentation based on data sensitivity

Safeguards 🔒

3.1 Establish and Maintain a Data Management Process

  • Address data sensitivity, ownership, handling
  • Define retention limits and disposal requirements
  • Annual review and updates

3.2 Establish and Maintain a Data Inventory

  • Focus on sensitive data at minimum
  • Annual review and updates

3.3 Configure Data Access Control Lists

  • Based on need-to-know
  • Apply to files, databases, and applications

3.4 Enforce Data Retention

  • Define minimum and maximum timelines
  • Follow enterprise data management process

3.5 Securely Dispose of Data

  • Follow enterprise disposal process
  • Match disposal method to data sensitivity

3.6 Encrypt Data on End-User Devices

  • Focus on sensitive data
  • Use tools like BitLocker, FileVault, dm-crypt

3.7 Establish and Maintain a Data Classification Scheme

  • Define classification labels
  • Annual review and updates

3.8 Document Data Flows

  • Include service provider flows
  • Annual review and updates

3.9 Encrypt Data on Removable Media

  • Protect all removable media
  • Ensure encryption before data transfer

3.10 Encrypt Sensitive Data in Transit

  • Use TLS, OpenSSH, etc.
  • Ensure secure transmission protocols

3.11 Encrypt Sensitive Data at Rest

  • Server-side encryption minimum
  • Consider client-side encryption

3.12 Segment Data Processing and Storage Based on Sensitivity

  • Separate processing environments by sensitivity
  • Maintain data segregation

3.13 Deploy a Data Loss Prevention Solution

  • Use automated DLP tools
  • Cover stored, processed, and transmitted data

3.14 Log Sensitive Data Access

  • Track modifications and disposal
  • Monitor access patterns

Secure Configuration of Enterprise Assets and Software

IG1 IG2 IG3
Total Safeguards 12
Asset Type Applications, Devices, Users
Security Function Protect, Respond

Overview 📋

Establish and maintain the secure configuration of enterprise assets (end-user devices, including portable and mobile; network devices; non-computing/IoT devices; and servers) and software (operating systems and applications).

Why is this Control critical? 🤔

  • Default configurations prioritize ease-of-deployment over security:
    • Basic controls often disabled
    • Open services and ports
    • Default accounts/passwords
    • Pre-configured DNS settings
    • Older vulnerable protocols
    • Pre-installed unnecessary software
  • Configuration updates need ongoing management throughout asset lifecycle
  • Changes must be tracked and approved through workflow process
  • Critical for on-premises, remote devices, and cloud environments
  • Service providers often use insecure default configurations
  • Configurations can degrade as software is updated or patched

Procedures and Tools 🔧

  • Start with publicly developed security baselines:
    • CIS Benchmarks™
    • NIST National Checklist Program
  • Steps to build secure baseline image:
    • Determine data risk classification
    • Create security configuration script
    • Install base OS
    • Apply OS patches
    • Install applications
    • Apply application updates
    • Install local customization scripts
    • Run security script
    • Record/score system settings
    • Perform security QA test
    • Save base image securely
  • Use configuration management tools:
    • CIS-CAT for measurement
    • Agent or agentless inspection
    • Regular configuration checks
    • Deviation detection

Safeguards 🔒

4.1 Establish and Maintain a Secure Configuration Process

  • Cover all enterprise assets and software
  • Annual review and updates
  • Document all procedures

4.2 Establish and Maintain a Secure Configuration Process for Network Infrastructure

  • Network-specific configurations
  • Annual review and updates
  • Regular assessment

4.3 Configure Automatic Session Locking

  • 15 minutes max for general systems
  • 2 minutes max for mobile devices
  • Enforce across all assets

4.4 Implement and Manage a Firewall on Servers

  • Virtual firewall options
  • OS firewall configuration
  • Third-party firewall agents

4.5 Implement and Manage a Firewall on End-User Devices

  • Default-deny rule
  • Port filtering
  • Allow only necessary services

4.6 Securely Manage Enterprise Assets and Software

  • Use version-controlled-infrastructure-as-code
  • Secure administrative interfaces
  • Use SSH and HTTPS

4.7 Manage Default Accounts

  • Disable or make unusable
  • Handle root/admin accounts
  • Manage vendor accounts

4.8 Uninstall or Disable Unnecessary Services

  • Remove unused services
  • Disable unnecessary functions
  • Regular review of services

4.9 Configure Trusted DNS Servers

  • Use enterprise-controlled DNS
  • Configure reputable external DNS
  • Regular DNS validation

4.10 Enforce Automatic Device Lockout

  • 20 attempts max for laptops
  • 10 attempts max for mobile devices
  • Implement device restrictions

4.11 Enforce Remote Wipe Capability

  • Enable for enterprise devices
  • Configure for lost/stolen scenarios
  • Test wipe functionality

4.12 Separate Enterprise Workspaces on Mobile Devices

  • Use configuration profiles
  • Implement work profiles
  • Separate enterprise/personal data

Account Management

IG1 IG2 IG3
Total Safeguards 6
Asset Type Users
Security Function Identify, Protect, Respond

Overview 📋

Use processes and tools to assign and manage authorization to credentials for user accounts, including administrator accounts, as well as service accounts, to enterprise assets and software.

Why is this Control critical? 🤔

  • Easier for attackers to use valid credentials than "hacking":
    • Weak passwords
    • Lingering accounts after user departure
    • Dormant test accounts
    • Shared accounts with unchanged passwords
    • Service accounts with embedded credentials
    • Password reuse from compromised accounts
    • Social engineering
    • Malware capturing credentials
  • Administrative accounts are prime targets:
    • Allow adding other accounts
    • Can make system changes
    • Higher risk of system compromise
  • Service accounts require special attention:
    • Often shared among teams
    • May be unknown until audits
    • Critical for operations
  • Account logging and monitoring is essential for:
    • Security operations
    • Incident detection
    • Compliance requirements
    • Access tracking

Procedures and Tools 🔧

  • Credential management:
    • Track credentials as assets
    • Implement password policies
    • Regular password updates
    • Secure storage methods
  • Account maintenance:
    • Regular account audits
    • Disable dormant accounts
    • Remove unnecessary accounts
    • Track new account creation
  • Authentication best practices:
    • Implement SSO where possible
    • Use password managers
    • Enable MFA for remote access
    • Automatic session timeout
  • Administrative access:
    • Separate admin accounts
    • Limited use for sensitive tasks
    • Regular activity monitoring
    • Privileged access management

Safeguards 🔒

5.1 Establish and Maintain an Inventory of Accounts

  • Include user and administrator accounts
  • Record name, username, start/stop dates, department
  • Quarterly validation minimum

5.2 Use Unique Passwords

  • 8+ characters with MFA
  • 14+ characters without MFA
  • Enforce complexity requirements

5.3 Disable Dormant Accounts

  • 45 days inactivity threshold
  • Regular account review
  • Automated monitoring

5.4 Restrict Administrator Privileges

  • Dedicated admin accounts
  • Separate accounts for regular activities
  • Limited administrative access

5.5 Establish and Maintain an Inventory of Service Accounts

  • Document department owner
  • Include review date and purpose
  • Quarterly validation minimum

5.6 Centralize Account Management

  • Use directory services
  • Implement identity services
  • Unified access control

Total Safeguards 8

Security Function Protect

Overview 📋

Use processes and tools to create, assign, manage, and revoke access credentials and privileges for user, administrator, and service accounts for enterprise assets and software.

Why is this Control critical? 🤔

  • Access management fundamentals:
    • Ensures appropriate access levels
    • Follows least privilege principle
    • Protects sensitive data
    • Enforces role-based access
  • High-risk activities requiring protection:
    • Access from untrusted networks
    • Administrator functions
    • Account management capabilities
    • System configuration changes
  • Common access control issues:
    • Excessive access rights
    • Lingering access from role changes
    • Unnecessary local admin privileges
    • Improper role-based access
  • Critical security requirements:
    • Strong authentication methods
    • Multi-factor authentication
    • Privileged access management
    • Regular access reviews

Procedures and Tools 🔧

  • Role-based access implementation:
    • Based on enterprise role
    • Following need-to-know principle
    • Consider privacy requirements
    • Enforce separation of duties
  • Authentication requirements:
    • Universal MFA for privileged accounts
    • Secure smartphone MFA applications
    • Number-generator over SMS/push
    • Strong password policies
  • Privileged access management:
    • PAM tool implementation
    • One-time password systems
    • "Jump-box" architecture
    • Out-of-band administration
  • Access lifecycle management:
    • Consistent de-provisioning
    • Contractor access control
    • Service account tracking
    • Regular access reviews

Safeguards 🔒

6.1 Establish an Access Granting Process

  • Automated process preferred
  • Cover new hires and role changes
  • Document approval workflow

6.2 Establish an Access Revoking Process

  • Immediate account disabling
  • Preserve audit trails
  • Automated revocation

6.3 Require MFA for Externally-Exposed Applications

  • Regular compliance checks

6.4 Require MFA for Remote Network Access

  • All remote access requires MFA
  • Secure authentication methods
  • Regular validation

6.5 Require MFA for Administrative Access

  • All admin accounts
  • On-site and third-party providers
  • Strong authentication methods

6.6 Establish and Maintain an Inventory of Authentication and Authorization Systems

  • Include on-site and remote systems
  • Annual review minimum
  • Regular updates

6.7 Centralize Access Control

  • Use directory services
  • Implement SSO where possible
  • Unified management

6.8 Define and Maintain Role-Based Access Control

  • Document necessary access rights
  • Annual review minimum
  • Validate all privileges

    • Vulnerabilities known in closed communities
    • Need for temporary mitigations
    • Defense-in-depth approach
  • Enterprise-wide challenges:
    • Scale of remediation efforts
    • Conflicting business priorities
    • Resource constraints
    • Testing requirements

Procedures and Tools 🔧

    • CVE compatibility
    • CCE support
    • OVAL integration
    • CPE mapping
    • CVSS scoring
    • XCCDF compliance

    • Increased frequency for diverse assets

    • Change tracking
    • Regular updates
  • Remediation management:
    • Integration with ticketing systems
    • Progress tracking
    • SLA monitoring
    • Management reporting
    • Trend analysis
  • Prioritization factors:
    • CVSS base scores
    • Threat intelligence
    • Asset criticality
    • Exposure level
    • Business impact

  • Cover all applications
  • Version control
  • Dependency management

7.5 Perform Automated Vulnerability Scans of Internal Enterprise Assets

Collect, alert, review, and retain audit logs of events that could help detect, understand, or recover from an attack.

Why is this Control critical? 🤔

  • Incident response support:
    • Understanding attack extent
    • Determining attack timeline
    • Identifying accessed information
    • Detecting data exfiltration
    • Evidence preservation
  • Long-term value:
    • Support follow-up investigations
    • Detect long-term undetected attacks
    • Meet compliance requirements
    • Establish activity baselines
    • Pattern analysis

Procedures and Tools 🔧

  • Comprehensive logging strategy:
    • Enable native logging capabilities
    • Configure verbose logging where beneficial
    • Centralized log collection
    • Log rotation policies
    • Storage management
  • Critical log sources:
    • Firewalls and security appliances
    • Network infrastructure devices
    • Authentication systems
    • Remote access solutions
    • Operating systems
    • Applications and services
  • Log management practices:
    • Regular log reviews
    • Automated analysis
    • Alert configuration
    • Correlation capabilities
    • Retention policies
  • Security considerations:
    • Log integrity protection
    • Access control to logs
    • Secure transmission
    • Backup procedures
    • Chain of custody
